Versionamento

Git e GitHub controle de verso para iniciantes

Git e GitHub so duas tecnologias que todo desenvolvedor deve aprender, independentemente de sua rea. Se voc um desenvolvedor iniciante, pode pensar que esses dois termos significam a mesma coisa, mas so diferentes.

Este roteiro ajudar voc a entender o que Git e controle de verso, os comandos bsicos do Git que voc precisa conhecer, como voc pode usar seus recursos para aumentar a eficincia do seu trabalho e como estender esses recursos usando o GitHub.

Este roteiro voltado a iniciantes, pois os exemplos sero muito fceis de entender. Tambm ser um tutorial generalizado para que qualquer pessoa possa acompanhar, no importa qual seja sua linguagem de programao favorita.

Para nosso projeto, faremos uma lista de tarefas escrita em um arquivo de texto (txt). Voc ver como podemos usar os recursos do Git para trabalhar e criar uma verso final da lista.

Pr-requisitos

Para concluir este roteiro, voc precisar do seguinte:

  • Uma interface de linha de comando, um terminal.
  • Um editor de texto de sua escolha (usarei o VS Code).
  • Uma conta no GitHub.

O que Git?

O Git um sistema de controle de verso que permite rastrear as alteraes feitas em seus arquivos ao longo do tempo. Com o Git, voc pode reverter para vrios estados de seus arquivos (como se usasse uma mquina do tempo). Voc tambm pode fazer uma cpia do arquivo, fazer alteraes nessa cpia e mesclar essas alteraes na verso original.

Por exemplo, voc pode estar trabalhando na pgina de destino de um site e descobrir que no gosta da barra de navegao. Ao mesmo tempo, voc pode ficar apreensivo em comear a alterar seus componentes, porque o resultado pode ser pior.

Com o Git, voc pode criar uma cpia idntica desse arquivo e mexer na barra de navegao. Ento, quando estiver satisfeito com suas alteraes, poder mesclar a cpia ao arquivo original.

Voc no est limitado a usar o Git apenas para arquivos de cdigo-fonte voc tambm pode us-lo para acompanhar arquivos de texto ou at imagens. Isso significa que o Git no apenas para desenvolvedores qualquer um pode ach-lo til.

Como instalar o Git

Para usar o Git, voc precisa instal-lo em seu computador. Para fazer isso, voc pode baixar a verso mais recente no site da web oficial. Voc pode fazer o download para o seu sistema operacional a partir das opes fornecidas.

Voc tambm pode instalar o Git usando a linha de comando, mas, como os comandos variam de acordo com cada sistema operacional, vamos nos concentrar na abordagem mais geral.

Como configurar o Git

Vou assumir que, neste ponto, voc j instalou o Git. Para verificar isso, voc pode executar este comando no terminal: git --version. Ele mostra a verso atual instalada no seu PC.

A prxima coisa que voc precisa fazer definir seu nome de usurio e endereo de e-mail. O Git usar essas informaes para identificar quem fez alteraes especficas nos arquivos.

Para definir seu nome de usurio, digite e execute estes comandos:

gitconfig --global user.name "SEU_NOME_DE_USUARIO"**

gitconfig --global user.email "SEU_E-MAIL"**.

Apenas certifique-se de substituir "SEU_NOME_DE_USUARIO" e "SEU_E-MAIL" pelos valores que voc escolher ou seja, seu nome de usurio e seu e-mail reais.

Como criar e inicializar um projeto no Git

Aps terminarmos de instalar e configurar o Git, agora hora de criar nosso projeto.

Criamos uma pasta, no desktop, chamada Git and GitHub tutorial. Usando a linha de comando, navegue at o local do seu novo projeto. Para execute os seguintes comandos:

Se voc conhece a linha de comando h pouco e ainda est aprendendo a us-la para navegar pelo seu PC, sugiro usar o Visual Studio Code da Microsoft. um editor de cdigo que possui um terminal embutido para executar comandos. Voc pode baix-lo aqui.

Depois de instalar o VS Code, abra seu projeto no editor e abra um novo terminal para seu projeto. Isso apontar automaticamente o terminal/linha de comando para o caminho do seu projeto.

Agora, para inicializar um repositrio do seu projeto, basta executar gitinit. Isso dir ao Git para comear a observar seus arquivos a cada alterao que ocorrer. Esta a aparncia na linha de comando/terminal:** **

A primeira linha tem informaes sobre meu PC e o caminho para a pasta. A segunda linha o comando gitinit e a terceira linha a resposta enviada de volta me dizendo que meu repositrio (ou repo) foi inicializado. Ele considerado vazio porque no informamos ao Git quais arquivos rastrear.** **

O que o GitHub?

O GitHub um servio de hospedagem on-line para repositrios do Git. Imagine trabalhar em um projeto em casa e, enquanto estiver fora talvez na casa de um amigo, de repente se dar conta da soluo para um erro de cdigo que o deixou inquieto por dias.

Voc no pode fazer essas alteraes porque seu PC no est com voc. Se, contudo, voc tiver seu projeto hospedado no GitHub, poder acessar e baixar esse projeto com um comando em qualquer computador ao qual tenha acesso. Em seguida, voc pode fazer suas alteraes e enviar a verso mais recente de volta ao GitHub.

Em resumo, o GitHub permite que voc armazene seu repositrio em sua plataforma. Outro recurso incrvel que vem com o GitHub a capacidade de colaborar com outros desenvolvedores de qualquer local.

Se voc iniciante, encontrar alguns termos novos como push, commit, adde assim por diante mas no se deixe assustar com eles. Com alguma prtica, voc ser capaz de lembrar desses termos e do que eles fazem.

Dividimos esta seo em etapas para ajud-lo a entender o processo com mais clareza.

Para poder usar o GitHub, ter de criar uma conta primeiramente. Voc pode fazer isso no[site da web](https://github.com/) do GitHub.

Passo 2 Crie um repositrio

Passo 3 Adicionar e confirmar arquivos

Antes de "adicionar" e "confirmar" nossos arquivos, voc precisa entender os estgios de um arquivo que est sendo rastreado peloGit.

Estado confirmado (committed)

Um arquivo est no estado confirmado quando todas as alteraes feitas no arquivo foram salvas no repositrio local. Os arquivos no estgio confirmado so arquivos prontos para serem enviados para o repositrio remoto (no GitHub).

Estado modificado (modified)

Um arquivo no estado modificado tem algumas alteraes feitas nele, mas ainda no foi salvo. Isso significa que o estado do arquivo foi alterado de seu estado anterior no estado confirmado.

Estado preparado (staged)

Um arquivo no estado preparado significa que est pronto para ser confirmado. Nesse estado, todas as alteraes necessrias foram feitas. Portanto, o prximo passo mover o arquivo para o estado de confirmao.

Voc pode entender isso melhor imaginando o Git como uma cmera. A cmera s tirar um instantneo quando o arquivo atingir o estado de confirmao. Aps este estado, a cmera comea a comparar as alteraes feitas no mesmo arquivo com o ltimo instantneo (este o estado modificado). Quando as alteraes necessrias forem feitas, o arquivo preparado e movido para o estado de confirmao para um novo instantneo.

Isso pode ser muita informao para absorver no momento, mas no desanime fica mais fcil com a prtica.

Como adicionar arquivos ao Git

Quando inicializamos nosso projeto, o arquivo no estava sendo rastreado pelo Git. Para isso, usamos o comando gitadd . O ponto que vem depois de add representa todos os arquivos que existem no repositrio. Se voc quiser adicionar um arquivo especfico (por exemplo, um arquivo chamado about.txt), use gitadd about.txt.

Agora, nosso arquivo est no estado preparado. Voc no receber uma resposta aps este comando, mas, para saber em que estado seu arquivo est, voc pode executar o comando git status. ** **

Como confirmar (commit) arquivos no Git

O prximo estado de um arquivo aps o estado preparado o estado confirmado. Para confirmar nosso arquivo, usamos o comando gitcommit -m "firstcommit"

A primeira parte do comando gitcommit diz ao Git que todos os arquivos preparados esto prontos para serem confirmados. Ento, hora de tirar um instantneo. A segunda parte, -m "firstcommit", a mensagem de confirmao. -m uma abreviao de mensagem enquanto o texto entre aspas a mensagem de confirmao (que pode ser a mensagem que voc quiser e no idioma que quiser).

Depois de executar este comando, voc deve obter uma resposta semelhante a esta:

gitcommit

Agora, nosso arquivo est no estado confirmado.

O segundo comando, gitbranch -M main, altera o nome do seu branch principal para "main". O branch padro pode ser criada como "master", mas "main" o nome padro para este repositrio agora. Geralmente, no h resposta aqui.** **
